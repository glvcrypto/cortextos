## P0 #4b — PR #246 round-3 fix (boss URGENT 04:04 UTC May 19)

### Dispatch
Boss URGENT (`1779163481511-boss-1mx90`): after b149118 push, the "headline" warning was fixed but a NEW "Undefined array key 'heading'" surfaced at product-category.php lines 312/317/319.

### Root cause
The brand-archive routing actually goes through product-category.php, NOT my thin-pointer boats-princecraft.php. `setup-pages.php:72` originally seeded the Princecraft WP page with `template = page-templates/product-category.php` (back in April). The new "Boats. Princecraft" template (Template Name registration in boats-princecraft.php) never executes because the WP admin page-template assignment was never updated.

product-category.php loops `foreach ($subcategories as $j => $tab)` and accesses `$tab['heading']`. princecraft config's `subcategories` use `{label, anchor}` not `{heading}` → undefined-key warnings.

### Shipped (commit `c5168b4` on `feat/brand-archive-series-showcase`)

Single change to `page-templates/product-category.php` (+13 lines), at the top after config loads and before any rendering:

```php
if ( ! empty( $config['series_models'] ) ) {
    require __DIR__ . '/series-showcase.php';
    return;
}
```

Architectural fix, not band-aid: any config declaring `series_models` triggers delegation to the series-showcase engine. Resolves the immediate warnings without per-page WP-admin template reassignment, AND auto-applies to every future brand archive (Mercury, Cub Cadet, Echo, Hisun, Toro) without DB work.

Header double-call avoided: product-category get_header is at line 250 (well after the new delegation return at ~line 50); series-showcase has its own get_header.

### PR #246 commit chain
1. `9d0ced1` — initial generalization + princecraft config wiring
2. `b149118` — brand_band wp_parse_args hardening + image fallback chain + headline keys
3. `c5168b4` — product-category.php delegates to series-showcase for series_models configs

Awaiting Aiden re-test via pr-test-local.bat + merge.

### Banked pattern
Brand-archive page-template assignment is a routing-mismatch class: setup-pages.php seeded ALL brand pages with product-category.php template. The thin-pointer pattern fails when the WP admin page-template assignment isn't updated. Execution-time delegation (config-driven) is the durable fix because it doesn't require DB migration.
