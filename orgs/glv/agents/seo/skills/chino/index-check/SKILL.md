# /chino:index-check — Indexation Health Check

> Checks the health of a client's indexation by cross-referencing GSC sitemap data with actual indexed pages. Identifies pages at risk and recommends content refresh or pruning.

## Trigger

- `/chino:index-check <client>`
- "Check indexation for <client>"
- "Index health for <client>"

## Autonomy

**Assignment:** `agent-autonomous` — read-only GSC data pulling + analysis.

---

## Intake

| Input | Required | Source |
|-------|----------|--------|
| `client` | Yes | Command parameter — resolved via alias map |

---

## Process

### Step 1: Resolve Client Config

1. Resolve client alias
2. Read `projects/<client>/CONTEXT.md` → extract `seo:` block
3. Get `gsc_property` and `output_path`

### Step 2: Load Knowledge Base

Read `.claude/skills/chino/references/6-level-framework.md` — focus on Level 5 (Indexed URLs) for assessment criteria:
- 76% indexation threshold
- Status progression: indexed → crawled-not-indexed → discovered-not-indexed → limbo
- Content bloat detection criteria (30-40% not-indexed = alert)

### Research Integration (Optional)

Before starting, check if `projects/<client>/research/` exists:

1. If `pillar-b-direct-competitors.md` exists → read it. Compare the client's indexation rate against competitor page counts identified in research — context for whether the client's index coverage is competitive.

Use research insights to enrich output. Cite specific findings when they inform recommendations. If research files don't exist, proceed normally — all core functionality works without research.

### Step 3: Pull GSC Data

Use the GSC MCP tools sequentially:

1. **List sitemaps:**
   ```
   mcp__google-search-console__list_sitemaps
   siteUrl: <gsc_property>
   ```

2. **Get each sitemap** (if the tool supports it):
   ```
   mcp__google-search-console__get_sitemap
   siteUrl: <gsc_property>
   path: <sitemap_url>
   ```

3. **Enhanced search analytics** (page dimension to see which pages have impressions):
   ```
   mcp__google-search-console__enhanced_search_analytics
   siteUrl: <gsc_property>
   startDate: 90 days ago
   endDate: today
   dimensions: page
   rowLimit: 5000
   ```

4. **Index inspection** for key pages (homepage + top service/product pages):
   ```
   mcp__google-search-console__index_inspect
   siteUrl: <gsc_property>
   inspectionUrl: <page_url>
   ```
   Run for up to 10 key pages identified from sitemap or search analytics.

### Step 4: Cross-Reference and Analyse

1. Compare sitemap URLs with pages that have impressions in search analytics
2. Identify pages in sitemaps that have zero impressions (potentially not indexed)
3. Use index inspection results to categorize pages by status
4. Calculate indexation coverage percentage
5. Flag pages that appear to be not indexed for 90+ days
6. Check for content bloat (combined not-indexed > 30-40% of total)

### Step 5: Save Output

Save to: `projects/<client>/seo/index-tracking/YYYY-MM-DD-index-check.md`

### Step 6: Update State

Update `.state/chino-state.json`:
- Set `lastIndexCheck` to today's date
- Append to `runs` array
