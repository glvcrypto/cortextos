# /chino:launch-checklist — SEO Launch Readiness

> Comprehensive SEO readiness check for new clients. Runs through all 6 levels of the framework with automated checks where possible and manual verification items where not. Produces a pass/fail checklist with specific fix instructions.

## Trigger

- `/chino:launch-checklist <client>`
- "SEO launch check for <client>"
- "Is <client> ready for SEO?"

## Autonomy

**Assignment:** `agent-autonomous` — read-only checks + reporting. Produces a checklist, does not make changes.

---

## Intake

| Input | Required | Source |
|-------|----------|--------|
| `client` | Yes | Command parameter — resolved via alias map |

---

## Process

### Step 1: Resolve Client Config

1. Resolve client alias
2. Read `projects/<client>/CONTEXT.md` → extract `seo:` block
3. Verify the config block exists and all fields are populated
4. Check that the client's SEO folders exist (`projects/<client>/seo/{audits,keyword-research,...}`)
5. Check that the client has an entry in `.state/chino-state.json`

### Step 2: Load Knowledge Base

Read `.claude/skills/chino/references/6-level-framework.md` for comprehensive checklist criteria across all 6 levels.

### Research Integration (Optional)

Before starting, check if `projects/<client>/research/` exists:

1. If `pillar-e-market-sizing.md` exists → read it. Add niche-specific compliance or regulatory items to the checklist (e.g., industry-specific disclaimers, licensing requirements).
2. If `pillar-g-community-local.md` exists → read it. Verify local directory coverage beyond the standard set — check for niche-specific directories or local media listings.
3. If `pillar-f-supplier-manufacturer.md` exists → read it. Check if manufacturer dealer locator listings are correct and if any co-op marketing requirements affect the site launch.

Use research insights to enrich output. Cite specific findings when they inform recommendations. If research files don't exist, proceed normally — all core functionality works without research.

### Step 3: Run Automated Checks

#### Level 1: External Presence (partially automated)

- **WebSearch:** Search for "[business name] sault ste marie" — check if GBP appears
- **WebSearch:** Search for "[domain]" — check if brand search shows knowledge panel or sitelinks
- **WebFetch:** Check for social media links on the website footer/header

#### Level 2: Domain Foundation (automated)

- **WebFetch:** Hit `https://<domain>/` — check SSL, response code, redirect chain
- **GSC:** Verify property is accessible: `mcp__google-search-console__list_sites`
- **GA4:** If configured, verify property returns data (quick runReport)
- **WebFetch:** Check DNS headers for Cloudflare

#### Level 3: Navigation & Architecture (automated)

- **WebFetch:** Analyse homepage for navigation structure, click depth to key pages
- **GSC:** `mcp__google-search-console__list_sitemaps` — check sitemaps are submitted
- **WebFetch:** Check `robots.txt` exists and is properly configured
- **WebFetch:** Check mobile viewport meta tag

#### Level 4: Crawlability (automated)

- **WebFetch:** Parse robots.txt for blocking rules
- **GSC:** Check for crawl errors
- **WebFetch:** Check canonical tags on homepage
- **WebFetch:** Check for noindex tags on key pages

#### Level 5: Indexation (automated)

- **GSC:** `enhanced_search_analytics` with page dimension — count indexed pages
- **GSC:** `list_sitemaps` — compare submitted vs indexed
- **Check for duplicate content signals

#### Level 6: On-Page (automated)

- **WebFetch:** Analyse homepage: title tag, H1, meta description, schema markup
- **WebFetch:** Check for structured data (JSON-LD)
- **WebFetch:** Check for llms.txt
- **WebFetch:** Check page speed indicators (viewport, render approach)

### Step 4: Compile Checklist

Score each item as:
- `[x]` — Passed (verified working)
- `[ ]` — Failed (needs fix)
- `[~]` — Partial (partially configured, needs improvement)
- `[?]` — Cannot verify (needs manual check)

### Step 5: Save Output

Save to: `projects/<client>/seo/audits/YYYY-MM-DD-launch-checklist.md`
