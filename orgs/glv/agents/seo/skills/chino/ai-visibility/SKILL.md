# /chino:ai-visibility — AI Bot Crawl & Referral Tracking

> Tracks AI visibility by analysing AI referral traffic in GA4, checking robots.txt for AI bot directives, and verifying llms.txt configuration. Based on Episode 6's AIO/GEO framework.

## Trigger

- `/chino:ai-visibility <client>`
- "AI visibility check for <client>"
- "Check AI referrals for <client>"

## Autonomy

**Assignment:** `agent-autonomous` — read-only data analysis.

---

## Intake

| Input | Required | Source |
|-------|----------|--------|
| `client` | Yes | Command parameter — resolved via alias map |

## Process

### Step 1: Resolve Client Config

1. Resolve client alias
2. Read `projects/<client>/CONTEXT.md` → extract `seo:` block
3. Note `ga4_mcp` (may be null) and `domain`

### Step 2: Load Knowledge Base

Read `.claude/skills/chino/references/6-level-framework.md` — focus on Level 6 AI visibility section:
- AI bot crawl monitoring via log files
- AI referral tracking via GA4 source/medium
- Known AI referral sources

### Research Integration (Optional)

Before starting, check if `projects/<client>/research/` exists:

1. If `pillar-d-global-intelligence.md` exists → read it. Note how best-in-class sites in this niche are handling AI visibility — are leaders in the space optimizing for AI citation, and what patterns are emerging?

Use research insights to enrich output. Cite specific findings when they inform recommendations. If research files don't exist, proceed normally — all core functionality works without research.

### Step 3: Check GA4 for AI Referral Traffic (if available)

If `ga4_mcp` is not null, use the client's GA4 MCP server to check for AI referral sources.

Run a report filtering for known AI sources:

```
<ga4_mcp>__runReport
startDate: 90 days ago
endDate: today
metrics: [{"name": "sessions"}, {"name": "activeUsers"}, {"name": "screenPageViews"}]
dimensions: [{"name": "sessionSource"}, {"name": "sessionMedium"}]
```

From the results, filter for AI-related sources:
- `chatgpt` / `chat.openai.com`
- `perplexity` / `perplexity.ai`
- `claude` / `claude.ai`
- `copilot` / `bing.com/chat`
- `searchwave`
- `you.com`
- `phind`
- Any other source containing "ai" or "chat" in the referral

Also run a page-level report to see which pages receive AI referral traffic:

```
<ga4_mcp>__runReport
startDate: 90 days ago
endDate: today
metrics: [{"name": "sessions"}]
dimensions: [{"name": "pagePath"}, {"name": "sessionSource"}]
dimensionFilter: {
  "filter": {
    "fieldName": "sessionSource",
    "stringFilter": {
      "matchType": "CONTAINS",
      "value": "chat"
    }
  }
}
```

### Step 4: Check robots.txt for AI Bot Directives

Use WebFetch:

```
WebFetch: https://<domain>/robots.txt
Prompt: "Parse this robots.txt and specifically identify directives for these AI bots:
- GPTBot (OpenAI's crawler)
- ClaudeBot / anthropic-ai (Anthropic's crawler)
- PerplexityBot (Perplexity's crawler)
- CCBot (Common Crawl, used by many AI companies)
- Google-Extended (Google AI training crawler)
- FacebookBot (Meta AI)
- Bytespider (ByteDance/TikTok AI)
For each bot, state whether it is: Explicitly allowed, Explicitly blocked, or Not mentioned (defaults to allowed).
Also note the general Disallow rules that would affect all bots."
```

### Step 5: Check llms.txt Configuration

```
WebFetch: https://<domain>/llms.txt
Prompt: "Read this file and summarize its contents. If it returns a 404 or error, note that llms.txt is not configured. If it exists, check: Is it well-structured? Does it describe the business? Does it list key pages? Does it include contact information?"
```

Also check for `llms-full.txt` if the basic one exists:

```
WebFetch: https://<domain>/llms-full.txt
Prompt: "Read this file. If it exists, summarize the additional content compared to llms.txt."
```

### Step 6: Compile AI Visibility Report

Combine all findings into a comprehensive report.

### Step 7: Save Output

Save to: `projects/<client>/seo/audits/YYYY-MM-DD-ai-visibility.md`

### Step 8: Update State

Append to `.state/chino-state.json` runs array.
