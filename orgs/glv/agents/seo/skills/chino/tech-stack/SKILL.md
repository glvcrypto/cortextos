# /chino:tech-stack — Technical Stack Snapshot

> Catalogues a client's full technical stack by analysing their website, checking for common tags and tools, and verifying SEO configuration. Based on Episode 7's tech stack cataloguing approach.

## Trigger

- `/chino:tech-stack <client>`
- "Tech stack for <client>"
- "Check technical setup for <client>"

## Autonomy

**Assignment:** `agent-autonomous` — read-only web analysis + reporting.

---

## Intake

| Input | Required | Source |
|-------|----------|--------|
| `client` | Yes | Command parameter — resolved via alias map |

---

## Process

### Step 1: Resolve Client Config

1. Resolve client alias
2. Read `projects/<client>/CONTEXT.md` → extract `seo:` block
3. Get `domain`, `gsc_property`, `output_path`

### Research Integration (Optional)

Before starting, check if `projects/<client>/research/` exists:

1. If `pillar-d-global-intelligence.md` exists → read it. Note tech trends in the niche (e.g., configurators, VR tours, 3D viewers) that the client's stack should support or prepare for.
2. If `pillar-f-supplier-manufacturer.md` exists → read it. Check if manufacturer-required tracking pixels, dealer portal integrations, or co-op marketing tags should be installed.

Use research insights to enrich output. Cite specific findings when they inform recommendations. If research files don't exist, proceed normally — all core functionality works without research.

### Step 2: Analyse Website

Use WebFetch to check the client's website. Run these checks:

1. **Homepage analysis:**
   ```
   WebFetch: https://<domain>/
   Prompt: "Analyse this page. Extract: page title, meta description, H1 tags, canonical URL, any analytics tags (GA4/GTM IDs), meta robots directives, schema markup present, Open Graph tags, viewport meta tag, language/locale settings. Also note the CMS/framework if detectable from page source indicators."
   ```

2. **robots.txt check:**
   ```
   WebFetch: https://<domain>/robots.txt
   Prompt: "Parse this robots.txt file. List all User-agent blocks, Disallow rules, Allow rules, Sitemap references, and any AI bot directives (GPTBot, ClaudeBot, PerplexityBot, etc.). Flag any potential issues."
   ```

3. **Sitemap check:**
   ```
   WebFetch: https://<domain>/sitemap.xml
   Prompt: "Parse this sitemap. Count the total number of URLs listed. Note the last modification dates. List the first 20 URLs. Note if it's a sitemap index pointing to other sitemaps."
   ```

4. **llms.txt check:**
   ```
   WebFetch: https://<domain>/llms.txt
   Prompt: "Read this llms.txt file and summarize its contents. If it returns a 404, note that llms.txt is not configured."
   ```

5. **Security headers check:**
   ```
   WebFetch: https://<domain>/
   Prompt: "Check for security-related indicators: HTTPS active, any CSP references, HSTS indicators."
   ```

### Step 3: Pull Semrush Data (if available)

If Semrush MCP is available, pull technical data:
- Site audit results (if project exists)
- Technology detection

### Step 4: Check GSC Status

Use GSC tools to verify:
- Property is verified and accessible
- Sitemaps are submitted
- Any crawl issues flagged

### Step 5: Compile Report

Combine all findings into the tech stack snapshot.

### Step 6: Save Output

Save to: `projects/<client>/seo/tech-stack/YYYY-MM-DD-tech-stack.md`

### Step 7: Update State

Update `.state/chino-state.json`:
- Set `lastTechStack` to today's date
- Append to `runs` array
