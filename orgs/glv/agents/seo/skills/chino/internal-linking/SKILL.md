# /chino:internal-linking — Internal Link Audit

> Audits the internal linking structure of a client's site. Maps link relationships, finds orphan pages, checks anchor text quality, and applies the silo linking model from Episode 5.

## Trigger

- `/chino:internal-linking <client>`
- "Internal link audit for <client>"
- "Check internal links for <client>"

## Autonomy

**Assignment:** `agent-autonomous` — read-only analysis + recommendations.

| Input | Required | Source |
|-------|----------|--------|
| `client` | Yes | Command parameter — resolved via alias map |

## Process

### Step 1: Resolve Client Config

1. Resolve client alias
2. Read `projects/<client>/CONTEXT.md` → extract `seo:` block

### Step 2: Load Knowledge Base

Read `.claude/skills/chino/references/6-level-framework.md` — focus on Level 5 internal linking:
- Previous/next chain model (sequential linking between blog posts in a silo)
- Hub-spoke model (supporting pages link to hub, hub links to all supporting)
- Anchor text best practices (descriptive, keyword-relevant, varied)
- Orphan page risks

### Research Integration (Optional)

Before starting, check if `projects/<client>/research/` exists:

1. If `avatars/` folder exists → read `test-group-panel.md` and `avatars/*.md`. Validate that the silo structure maps to how avatars actually navigate — ensure link paths follow real customer journeys.
2. Check if a content plan exists at `projects/<client>/seo/content-plans/`. If so, cross-reference silo links with planned content to ensure internal linking supports the content roadmap.

Use research insights to enrich output. Cite specific findings when they inform recommendations. If research files don't exist, proceed normally — all core functionality works without research.

### Step 3: Get All Indexed Pages

```
mcp__google-search-console__enhanced_search_analytics
siteUrl: <gsc_property>
startDate: 90 days ago
endDate: today
dimensions: page
rowLimit: 5000
```

### Step 4: Crawl Key Pages for Links

For each key page (top 20 by impressions/clicks + all navigation pages), use WebFetch to extract internal links:

```
WebFetch: <page_url>
Prompt: "List ALL internal links on this page. For each link provide:
1. The destination URL (href)
2. The anchor text
3. Where on the page it appears (navigation, body content, footer, sidebar)
Only include links that point to the same domain. Return as a structured list."
```

Limit to 20 most important pages to keep the audit manageable.

### Step 5: Build Link Map

Create a matrix showing:
- Which pages link to which
- Anchor text used for each link
- Number of inbound internal links per page
- Number of outbound internal links per page

### Step 6: Identify Issues

1. **Orphan pages** — Pages with zero inbound internal links (not reachable from navigation or body content)
2. **Thin pages** — Pages with only 1-2 inbound links (fragile in the link graph)
3. **Over-linked pages** — Pages receiving disproportionate internal links (link equity dilution)
4. **Generic anchors** — Links using "click here", "read more", "learn more" instead of descriptive text
5. **Missing silo links** — Blog posts in the same topic that don't link to each other
6. **Missing hub links** — Supporting content that doesn't link back to its hub page

### Step 7: Generate Recommendations

For each issue, provide specific fix instructions:
- "Add link from [Page A URL] to [Page B URL] using anchor text '[suggested anchor]'"
- Format as actionable, copy-paste-ready instructions

### Step 8: Save Output

Save to: `projects/<client>/seo/internal-linking/YYYY-MM-DD-internal-linking.md`
