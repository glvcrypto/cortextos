## Slide 4: The four required Event schema fields

"Google requires four fields to be eligible for rich results:
1. **name** — event name
2. **startDate** — ISO 8601 format (e.g. 2026-12-14T10:00)
3. **location** — Place block with business name + full address
4. **eventStatus** — EventScheduled (update to Cancelled/Postponed if needed)"

"1. **endDate** — complete the time window
2. **description** — 1–2 sentences; include service or product type
3. **image** — real event photo, min 1200×675px
4. **offers** — price or Free; use priceCurrency: 'CAD' for Canadian businesses"

"1. Create a dedicated event page on your website (not just a GBP post — schema lives on the page)
2. Add Event schema in a `<script type='application/ld+json'>` block
3. Test with Google's Rich Results Test before publishing
4. Also create a GBP event post — separate from schema, reinforces local visibility"

## Slide 7: One practical note

## LinkedIn Caption (~460 words)

If your local business runs events — open houses, seasonal sales, workshops, holiday demos — Google can show them in search results before anyone clicks to your site.

Most local businesses have never set this up.

**What Google Events results look like:**

They're a separate rich result type that appears for event-related queries: "[business name] open house", "[service] workshop near me", "[product type] demo near me". They show the event name, date, location, and price directly in the search result — above organic listings.

You don't pay for placement. You set it up once with Event schema on a dedicated page on your website.

**Who this applies to:**

— Retailers running seasonal sales or open days
— Fitness studios, yoga studios, cooking schools with recurring sessions
— Home service businesses hosting client appreciation events or showroom days
— Nurseries, garden centres, and seasonal outdoor retailers
— Any business with a customer-facing event that has a specific date

**The four required fields:**

Event schema (JSON-LD format) needs four fields to qualify for rich results:
1. `name` — the event name
2. `startDate` — ISO 8601 format (e.g. 2026-12-14T10:00)
3. `location` — a Place block with business name + full address
4. `eventStatus` — EventScheduled (update to Cancelled or Postponed if plans change)

**Recommended additions that improve how the result displays:**

— `endDate` to show the full time window
— `description` with 1–2 sentences naming the service or product type
— `image` using a real event photo at least 1200×675px
— `offers` with price or Free; use priceCurrency: "CAD" for Canadian businesses

**Implementation in four steps:**

1. Create a dedicated event page on your website — the schema lives on the page, not just in a GBP post.
2. Add the Event schema in a `<script type="application/ld+json">` block.
3. Test with Google's Rich Results Test before publishing.
4. Create a GBP event post in parallel — it won't generate the rich result, but it reinforces local visibility.

**One practical note:**

Event schema is for events with specific dates — not ongoing promotions. If you run the same class every week, use separate schema per session or an EventSeries. Remove or update the eventStatus after the date passes — stale "upcoming event" schema with a past date is a quality signal Google reads.
